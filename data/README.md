# データ管理

このフォルダには、Webサイトで表示されるニュースとレポートのデータが保存されています。

## 📄 ファイル

### `news.json`
ニュース記事のデータを管理します。

**データ項目：**
- `id`: 一意の識別番号（重複しないように連番で付ける）
- `slug`: URL用の英数字ID（例: `website-renewal-2026`）
- `date`: 公開日（形式: `YYYY.MM.DD`）
- `category`: カテゴリー（`News`, `Event`, `Project`, `Press`, `Award`）
- `title`: タイトル
- `description`: 本文・説明文

**新しいニュースを追加する方法：**
1. `news.json` を開く
2. 配列の先頭に新しいオブジェクトを追加（最新が上）
3. `id` は既存の最大値+1
4. `slug` は一意の英数字（例: `new-service-launch-2026`）
5. 保存してサーバーを再起動

**例：**
```json
{
  "id": 7,
  "slug": "new-service-launch-2026",
  "date": "2026.03.01",
  "category": "News",
  "title": "新サービスをリリースしました",
  "description": "新しいサービスの説明文をここに記載します。"
}
```

### `reports.json`
研究レポートのデータを管理します。

**データ項目：**
- `id`: 一意の識別番号
- `title`: タイトル
- `client`: クライアント名
- `subtitle`: サブタイトル
- `tags`: タグの配列（例: `["#脳波", "#音楽"]`）
- `img`: 画像URL
- `featured`: トップページに表示するか（`true` / `false`）
- `category`: カテゴリー（`category1`, `category2`, etc.）
- `year`: 年度

**新しいレポートを追加する方法：**
1. `reports.json` を開く
2. 配列に新しいオブジェクトを追加
3. 画像は `/static/` フォルダに配置するか、外部URLを使用
4. 保存してサーバーを再起動

## 🔄 変更を反映する方法

1. JSONファイルを編集
2. ファイルを保存
3. サーバーを再起動（開発環境）
   ```bash
   npm run dev
   ```

## ⚠️ 注意事項

- JSONの構文エラーに注意（カンマ、括弧の閉じ忘れなど）
- `id` は重複しないこと
- `slug` も重複しないこと（URLになります）
- 日付は新しい順に並べると管理しやすい
- バックアップを取ることを推奨
